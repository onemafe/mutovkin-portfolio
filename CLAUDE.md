# Portfolio — Андрей Мутовкин

Личный сайт-портфолио (интерактивная 3D-визуализация архитектуры, Unreal Engine,
BIM). Перенесён с конструктора Tilda на чистую статику, чтобы можно было
постепенно дополнять новыми кейсами без привязки к платформе.

## Стек

Обычные статические **HTML / CSS / JS**. Никакой сборки, бандлера или
фреймворка — сайт открывается как есть. Так и должно оставаться, если не
появится явная причина усложнять.

## Архитектура

```
index.html      — вся страница (один файл, секции идут одна за другой)
css/
  project-style.css  — сгенерированный дизайн (цвета, шрифты, раскладка блоков)
  lib/               — самостоятельно захостенные библиотеки Tilda-рантайма
js/
  custom-fixes.js    — наши правки поверх Tilda-рантайма (см. ниже)
  project-script.js  — пустой (в оригинале не было кастомного JS проекта)
  lib/               — самостоятельно захостенные библиотеки Tilda-рантайма
images/           — все картинки/иконки
video/            — исходники видео (локально, в git не попадают, см. .gitignore)
scripts/          — служебные скрипты для загрузки видео на Kinescope (см. раздел «Видео»)
```

Сайт визуально и структурно воспроизводит оригинальный Tilda-сайт: HTML и CSS
взяты из фактического рендера редактора, а не переписаны вручную. Разметка
местами избыточна (Tilda генерирует много служебных классов/атрибутов) — это
осознанный компромисс ради 100% визуального соответствия оригиналу. Для
рантайма (анимации, видео-плеер, липкая кнопка «связаться» и т.д.)
используются самостоятельно захостенные JS/CSS библиотеки Tilda
(`js/lib`, `css/lib`) — без них вёрстка не заработает как надо.

`js/custom-fixes.js` — единственный наш собственный код. Сейчас там шим для
`t393_appearMenu` (плавающая кнопка «связаться»), потому что оригинальная
реализация грузится Tilda только на опубликованном сайте, а не в превью
редактора, откуда был сделан экспорт.

## Черновик реорганизации (mockup)

Сайт постепенно переезжает на новую структуру/дизайн (тёмная тема), которую
обкатываем в черновике, не трогая боевой `index.html`:

- `mockup.html` (корень) — текущий черновик реорганизации, дорабатывается
  прямо в этом репо.
- `Example/` — референсный дизайн (сторонний портфолио-сайт), на который
  ориентируется `mockup.html`. Не в git (`.gitignore`) — внутри чужие
  приватные материалы (резюме/сопроводительное письмо), только для
  локального визуального сравнения.
- `PortfolioAndrei/` — экспорт сессии из внешнего инструмента (Claude в
  дизайн-режиме), куда `mockup.html` носили доработать дизайн hero/about/
  experience. Содержит более свежую версию `mockup.html` (там же
  `concepts-video.html` — отдельный черновик-презентация с концептами
  автоплей-видео на главной, тоже не подключён к сайту) и новые ассеты в
  `images/` (кейсы `cases/`, логотипы, аватарки), которых ещё нет в
  основном `images/`. Папка — источник контента для переноса, сама по себе
  не часть сайта. Служебный мусор инструмента (`uploads/`, `screenshots/`,
  `v/`, `.thumbnail`) оттуда удалён после выгрузки — не восстанавливать.

Итоговый перенос — как обычно: руками скопировать нужные секции/ассеты в
`index.html` и `images/` (см. «Правила разработки» ниже), черновики не
заменяют `index.html` напрямую.

## Как запустить локально

Нужен любой статический сервер (открытие файла напрямую через `file://`
может сломать часть скриптов):

```
npx serve .
# или
python -m http.server 8080
```

## Видео

Все видео на сайте хостятся на **Kinescope** (`https://kinescope.io/embed/...`,
вставлены как `<iframe>`), а не лежат на сайте сами. Причина: YouTube с
02.2026 полностью заблокирован в России без VPN, Vimeo у российских
пользователей регулярно виснет/тормозит (CDN через Cloudflare зажат
Роскомнадзором), а VK Video блокирует доступ с иностранных IP. Kinescope —
российский сервис с серверами по всему миру, работает одинаково надёжно и в
России, и за границей.

Исходники mp4 лежат в `video/` только локально (в git не попадают — один файл
превышает лимит GitHub на 100 МБ/файл, см. `.gitignore`).

**Как добавить видео нового кейса:**
1. Положить `.mp4` файл в `video/`
2. `node scripts/upload-to-kinescope.js` — загрузит все новые файлы из `video/`
   на Kinescope (уже загруженные пропускает) и допишет ссылки в
   `scripts/kinescope-manifest.json`. Если файл большой (десятки–сотни МБ) и
   падает с `fetch failed` — загрузить его отдельно через `node
   scripts/upload-tus.js <имя_файла.mp4>` (загрузка по частям, надёжнее на
   медленном/нестабильном соединении).
3. Вставить в `index.html` `<iframe src="ЭМБЕД_ССЫЛКА_ИЗ_МАНИФЕСТА" ...>`
   по образцу уже вставленных на странице (ищи `kinescope.io/embed` в файле) —
   для фонового автоплей-видео нужны параметры `behaviour[autoPlay]=true` и
   т.п. (см. `scripts/apply-kinescope.js` для точного вида), для видео с
   элементами управления — просто ссылка без параметров.

Нужен токен Kinescope API в `.env` (`KINESCOPE_API_TOKEN=...`, права
«Загружать файлы» + «Читать все данные», без прав на удаление/полный доступ).

## Правила разработки

- Не добавлять сборку/фреймворк/бандлер без явного запроса — сайт должен
  оставаться простой статикой.
- Обязательна адаптивность под мобильные устройства для любых новых секций.
- Автотестов нет — новый кейс/секцию проверять глазами в браузере (десктоп +
  мобильная ширина).
- Новые кейсы добавляются прямо в `index.html` по образцу существующих
  секций (`<div class="r t-rec" ...>`), с копированием подходящего блока и
  правкой контента.
