# Portfolio — Андрей Мутовкин

Личный сайт-портфолио (интерактивная 3D-визуализация архитектуры, Unreal Engine,
BIM). Изначально перенесён с конструктора Tilda на чистую статику, затем
(2026-07) полностью переработан в новый дизайн (тёмная тема) — этот новый
`index.html` сейчас и есть боевой сайт.

## Стек

Обычные статические **HTML / CSS / JS**. Никакой сборки, бандлера или
фреймворка — сайт открывается как есть. Так и должно оставаться, если не
появится явная причина усложнять.

## Архитектура

```
index.html      — вся страница: один самодостаточный файл (инлайн CSS + JS,
                  контент кейсов/опыта — в JS-объекте DATA, рендерится в DOM
                  на лету). Единственная внешняя зависимость — шрифты Google
                  Fonts. Навигация по кейсам — через location.hash
                  (#company:marta и т.п.), см. функцию router().
images/           — все картинки/иконки (в т.ч. images/cases/, images/logos/,
                  images/photos/)
uploads/          — тоже реальный контент (скриншоты/рендеры кейсов),
                  на которые index.html ссылается напрямую как "uploads/...".
                  Исторически осталось из формата экспорта другого
                  инструмента — не мусор и не переносить в images/ без
                  правки путей в index.html.
video/            — исходники видео (локально, в git не попадают, см. .gitignore)
scripts/          — служебные скрипты для загрузки видео на Kinescope (см. раздел «Видео»)
css/, js/         — ОСИРОТЕВШИЕ остатки старого Tilda-сайта (до переработки
                  2026-07). Новый index.html их не подключает и не
                  использует. Оставлены как есть на момент переработки —
                  можно удалить, если явно попросят, но самостоятельно не
                  трогать.
```

## PortfolioAndrei/ — источник текущего дизайна

Новый `index.html` был доработан во внешнем инструменте (Claude в
дизайн-режиме) и выгружен обратно в проект как `PortfolioAndrei/mockup.html` +
ассеты. Это НЕ черновик рядом с сайтом — это и есть теперь сам сайт: контент
`PortfolioAndrei/mockup.html` был скопирован в корневой `index.html`
(2026-07-15), а нужные ассеты из `PortfolioAndrei/images/` и
`PortfolioAndrei/uploads/` перенесены в основные `images/` и `uploads/`.
Старый `mockup.html` в корне и Tilda-версия `index.html`, которые были до
этого, — заменены (история — в git).

В `PortfolioAndrei/` осталось:
- `concepts-video.html` — отдельный черновик-презентация с концептами
  автоплей-видео на главной, пока не подключён к сайту.

`Example/` — референсный дизайн (сторонний портфолио-сайт), на который
ориентировался `mockup.html` при доработке. Не в git (`.gitignore`) — внутри
чужие приватные материалы (резюме/сопроводительное письмо), только для
локального визуального сравнения.

**Важный урок (2026-07-15):** при выгрузке из внешнего инструмента папки типа
`uploads/`, `screenshots/`, `v/`, `.thumbnail` выглядят как служебный мусор
самого инструмента — но `uploads/` в этом экспорте оказалась реальным
хранилищем контента, на которое ссылается сам HTML. Прежде чем удалять
что-либо из выгрузки внешнего инструмента — обязательно грепнуть HTML на
ссылки на эту папку (`grep -o '"uploads/[^"]*"'` и т.п.), а не судить по
названию папки.

## Как запустить локально

Нужен любой статический сервер (открытие файла напрямую через `file://`
может сломать часть скриптов):

```
npx serve .
# или
python -m http.server 8080
```

## Видео

Все видео на сайте хостятся на **Kinescope** (`https://kinescope.io/embed/...`,
вставлены как `<iframe>`), а не лежат на сайте сами. Причина: YouTube с
02.2026 полностью заблокирован в России без VPN, Vimeo у российских
пользователей регулярно виснет/тормозит (CDN через Cloudflare зажат
Роскомнадзором), а VK Video блокирует доступ с иностранных IP. Kinescope —
российский сервис с серверами по всему миру, работает одинаково надёжно и в
России, и за границей.

Исходники mp4 лежат в `video/` только локально (в git не попадают — один файл
превышает лимит GitHub на 100 МБ/файл, см. `.gitignore`).

**Как добавить видео нового кейса:**
1. Положить `.mp4` файл в `video/`
2. `node scripts/upload-to-kinescope.js` — загрузит все новые файлы из `video/`
   на Kinescope (уже загруженные пропускает) и допишет ссылки в
   `scripts/kinescope-manifest.json`. Если файл большой (десятки–сотни МБ) и
   падает с `fetch failed` — загрузить его отдельно через `node
   scripts/upload-tus.js <имя_файла.mp4>` (загрузка по частям, надёжнее на
   медленном/нестабильном соединении).
3. Вставить в `index.html` `<iframe src="ЭМБЕД_ССЫЛКА_ИЗ_МАНИФЕСТА" ...>`
   по образцу уже вставленных на странице (ищи `kinescope.io/embed` в файле) —
   для фонового автоплей-видео нужны параметры `behaviour[autoPlay]=true` и
   т.п. (см. `scripts/apply-kinescope.js` для точного вида), для видео с
   элементами управления — просто ссылка без параметров.

Нужен токен Kinescope API в `.env` (`KINESCOPE_API_TOKEN=...`, права
«Загружать файлы» + «Читать все данные», без прав на удаление/полный доступ).

## Известные ограничения

- Формы обратной связи на сайте нет (убрана вместе с Tilda-версией) — только
  прямые контакты (email, Telegram, WhatsApp) в футере.

## Правила разработки

- Не добавлять сборку/фреймворк/бандлер без явного запроса — сайт должен
  оставаться простой статикой.
- Обязательна адаптивность под мобильные устройства для любых новых секций.
- Автотестов нет — новый кейс/секцию проверять глазами в браузере (десктоп +
  мобильная ширина). После любой правки в `index.html`, трогающей контент
  кейсов/ассеты, — грепнуть `"images/` и `"uploads/` пути на существование
  файлов и открыть страницу кейса (`#company:<id>`) в браузере, не только
  главный экран.
- Новые кейсы/опыт добавляются через объект `DATA` в `<script>` внутри
  `index.html` (массивы `experience`, `cases` и т.п.) — не через ручную
  разметку блоков, вёрстка генерируется JS-рендер-функциями
  (`renderMedia`, `renderCompany` и т.д.).
